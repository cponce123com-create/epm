Construye una plataforma web de periodismo ciudadano, opinión y denuncia llamada "El Príncipe Mestizo". 
Es el blog personal de un columnista independiente de San Ramón, Chanchamayo (Perú), que expone 
irregularidades en la gestión pública local. La estética debe ser editorial y periodística — 
similar a Medium o El País, pero con identidad propia: sobria, inteligente, con un toque de 
urgencia y denuncia.

═══════════════════════════════════════
  STACK TÉCNICO
═══════════════════════════════════════

Backend:
- Node.js + Express
- PostgreSQL (Neon) con Drizzle ORM
- JWT + bcrypt para autenticación
- Multer + Cloudinary SDK para subida de imágenes
- Slugify para generar slugs automáticos desde el título

Frontend:
- React 18 + Vite
- React Router v6
- Tailwind CSS
- TipTap (rich text editor con extensiones: Bold, Italic, Heading, BulletList, 
  OrderedList, Blockquote, Link, Image, HorizontalRule, Underline, TextAlign)
- Lucide React (íconos)
- date-fns (formateo de fechas en español)

Despliegue target: Render (monorepo)

═══════════════════════════════════════
  BASE DE DATOS (Drizzle ORM + Neon)
═══════════════════════════════════════

Tabla: users
- id (serial PK)
- email (varchar, unique)
- password_hash (varchar)
- display_name (varchar)  → "El Príncipe Mestizo"
- avatar_url (varchar, nullable)
- role (enum: 'admin' | 'author')
- created_at (timestamp)

Tabla: categories
- id (serial PK)
- name (varchar)           → Denuncia / Opinión / Investigación / Ciudad / Política
- slug (varchar, unique)
- color (varchar)          → color hex para badge (#C0392B, #2980B9, etc.)
- description (text)

Tabla: articles
- id (serial PK)
- title (varchar)
- slug (varchar, unique)
- summary (text)           → bajada del artículo, 2-3 oraciones
- cover_image_url (varchar, nullable)
- cover_image_alt (varchar, nullable)
- content (text)           → HTML generado por TipTap
- category_id (integer, FK → categories.id)
- author_id (integer, FK → users.id)
- status (enum: 'draft' | 'published')
- featured (boolean, default false)  → para destacar en home
- views (integer, default 0)
- reading_time (integer)   → minutos estimados (calcular al guardar: palabras/200)
- published_at (timestamp, nullable)
- created_at (timestamp)
- updated_at (timestamp)

Tabla: comments
- id (serial PK)
- article_id (integer, FK → articles.id)
- author_name (varchar)
- author_email (varchar)
- content (text)
- approved (boolean, default false)
- created_at (timestamp)

Tabla: site_settings
- id (serial PK)
- key (varchar, unique)    → 'about_text', 'adsense_client', 'site_description', etc.
- value (text)

Ejecutar migrate.js al iniciar el server si las tablas no existen.
Insertar categorías por defecto y usuario admin por defecto en el primer arranque:
- Admin: email desde env ADMIN_EMAIL, password desde env ADMIN_PASSWORD

═══════════════════════════════════════
  API ENDPOINTS (Express)
═══════════════════════════════════════

Públicos:
GET    /api/articles              → lista paginada (?page=1&limit=10&category=slug&search=texto)
GET    /api/articles/featured     → artículos con featured=true (máximo 3)
GET    /api/articles/:slug        → artículo individual + incrementar views
GET    /api/articles/:slug/related → 3 artículos de la misma categoría
GET    /api/categories            → todas las categorías
GET    /api/comments/:articleId   → comentarios aprobados de un artículo
POST   /api/comments              → crear comentario (queda pendiente de aprobación)
GET    /api/settings/public       → devuelve about_text, site_description, adsense_client

Protegidos (requieren JWT en header Authorization: Bearer <token>):
POST   /api/auth/login            → { email, password } → { token, user }
POST   /api/auth/logout
GET    /api/admin/articles        → todos los artículos (incluyendo borradores)
POST   /api/admin/articles        → crear artículo
PUT    /api/admin/articles/:id    → editar artículo
DELETE /api/admin/articles/:id    → eliminar artículo
PATCH  /api/admin/articles/:id/publish   → cambiar status a published
PATCH  /api/admin/articles/:id/feature   → toggle featured
GET    /api/admin/comments        → todos los comentarios (aprobados y pendientes)
PATCH  /api/admin/comments/:id/approve  → aprobar comentario
DELETE /api/admin/comments/:id    → eliminar comentario
POST   /api/upload/image          → subir imagen a Cloudinary, devuelve { url }
GET    /api/admin/settings        → todas las configuraciones
PUT    /api/admin/settings        → actualizar configuraciones (body: { key, value })
GET    /api/admin/stats           → { totalArticles, totalViews, pendingComments, totalComments }

- Slot de publicidad inline: insertarlo después del 3er párrafo automáticamente
- Bloque "Compartir en redes": botones para X/Twitter, Facebook y copiar enlace
- Sección de artículos relacionados (misma categoría, máximo 3 cards horizontales)
- Sección de comentarios al final:
  - Mostrar comentarios aprobados con nombre y fecha
  - Formulario: nombre, email (no se muestra públicamente), comentario
  - Mensaje post-envío: "Tu comentario está pendiente de moderación"
- Sidebar: slot publicidad, artículos más leídos

CATEGORÍA (/categoria/:slug):
- Banner con nombre de categoría y descripción
- Grid de artículos filtrados por esa categoría con paginación

BÚSQUEDA (/buscar?q=texto):
- Input de búsqueda en la parte superior con el término actual
- Resultados en lista: título, bajada, categoría, fecha
- Mensaje "No se encontraron resultados" si aplica

ACERCA DE (/acerca-de):
- Texto editable desde el admin (guardado en site_settings con key='about_text')

LOGIN (/admin/login):
- Formulario email + password
- Guardar token en localStorage
- Redirigir a /admin/dashboard tras login exitoso

DASHBOARD (/admin/dashboard):
- Tarjetas con métricas: Total artículos, Total vistas, Comentarios pendientes
- Tabla de últimos 5 artículos con estado y acciones rápidas

LISTA DE ARTÍCULOS (/admin/articles):
- Tabla con: portada (thumbnail), título, categoría, estado (badge), 
  vistas, fecha, acciones (editar, publicar/despublicar, destacar, eliminar)
- Filtros por estado (todos / publicados / borradores)
- Buscador por título

EDITOR DE ARTÍCULOS (/admin/articles/new y /admin/articles/:id/edit):
- Campo: Título (input grande)
- Campo: Bajada/Resumen (textarea)
- Selector de categoría (dropdown)

  previsualización inmediata de la imagen subida a Cloudinary
- Alt text de la imagen de portada
- Editor TipTap con toolbar completa:
  H1, H2, H3 | Negrita, Cursiva, Subrayado | Lista, Lista numerada | 
  Cita, Línea divisoria | Link | Insertar imagen (URL) | Alineación
- Botones: "Guardar borrador" y "Publicar"
- Auto-guardado cada 60 segundos si hay cambios (PATCH al backend)

COMENTARIOS (/admin/comments):
- Tabs: Pendientes / Aprobados / Todos
- Cada comentario muestra: nombre, email, artículo al que pertenece, 
  contenido, fecha, botones Aprobar y Eliminar

CONFIGURACIÓN (/admin/settings):
- Campo: Texto "Acerca de" (textarea con markdown básico)
- Campo: AdSense Publisher ID (ca-pub-XXXXXXXX)
- Campo: Descripción del sitio (para SEO)
- Campo: URL de Twitter/X
- Campo: URL de Facebook
- Botón Guardar
